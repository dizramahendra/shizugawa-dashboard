## Required map visualization logic

### 1. River layer = SWAT-style network visualization
The river layer should behave like a **hydrological network of reaches / tributaries**.

Use this visual rule set:

- treat rivers as a connected river network
- preserve hierarchy between:
  - main channel
  - secondary tributaries
  - smaller branches
- line weight / prominence can help communicate hierarchy
- show the selected variable as **time-varying color on river reaches / segments**
- each river should visibly change color as playback moves over time
- do not show each river as one flat static color category only
- keep it readable and productized, not raw engineering-software clutter

The river should feel like:
- a monitored network of reaches
- carrying variable values over time

Not like:
- decorative linework
- static classification only

### 2. Ocean basin = time-varying scalar field
The ocean basin should also respond to the selected variable and current timestep.

Use:
- the ocean basin geometry as the spatial container
- a simplified heatmap / scalar-field logic
- time-varying color change during playback
- maintain product readability and restraint

Do not keep the ocean basin as just one static blue fill.
